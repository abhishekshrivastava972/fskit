NSObject(FSKRequestDelegate)


Discussion



The FSKRequestDelegate category on NSObject defines FSKRequest delegate methods that can be implemented by objects to receive informational callbacks about the asynchronous load of a FamilySearch API request.

Note that all these will be called on the thread that started the asynchronous load operation on the associated FSKRequest object.

The following contract governs the delivery of the callbacks defined in this interface:

  • Before receiving a response or processing a redirect, request:didReceiveAuthenticationChallenge: may be received if authentication is required.
  • Unless a FSKRequest receives a -cancel message, the delegate will receive one and only one of -request:didReturnResponse:, or -request:didFailWithError: message, but never both. In addition, once either of these callbacks is sent, the delegate will receive no further callbacks of any kind for the given FSKRequest.

Methods

-request:didCancelAuthenticationChallenge:
Cancel authentication for a given request
-request:didFailWithError:
This method is called when an FSKRequest has failed to load successfully.
-request:didReceiveAuthenticationChallenge:
Start authentication for a given challenge
-request:didReturnResponse:
This method is called when an FSKRequest has finished loading successfully.
-windowForAuthenticationSheet:
Provides the window to attach the authentication sheet to

windowForAuthenticationSheet:

Provides the window to attach the authentication sheet to 

- (NSWindow *)windowForAuthenticationSheet:(FSKRequest *)request;
Discussion

If you implement this delegate method, you can choose which window will have the authentication sheet attached to it during an authentication challenge. If you do not implement this method, by default the sheet will attempt to attach to [.NSApp mainWindow] If you return nil, the authentication will be performed with an application modal dialog instead of a sheet.
